# Metadata Screenshots

This folder contains screenshots for the GLB Transform extension, showcasing the extension's features and user interface.

## Screenshots Included

According to Raycast extension guidelines, extensions with view commands require screenshots in the metadata folder. This folder includes screenshots for both view commands and an example of a no-view command in action.

### Screenshot Files:

#### Advanced GLB Transform (View Command)
1. **advanced-glb-1.png** - Screenshot of the Advanced GLB Transform form interface showing the various transformation options available
2. **advanced-glb-2.png** - Screenshot demonstrating the Advanced GLB Transform in use with configuration options

#### Inspect GLB File (View Command)
3. **inspect-glb-1.png** - Screenshot of the Inspect GLB File detail view showing file analysis
4. **inspect-glb-2.png** - Screenshot displaying GLB inspection results with markdown-formatted analysis

#### No-View Command Example
5. **draco-compression.png** - Screenshot showing the Draco compression command workflow and HUD notification

## Technical Specifications

- **Image Dimensions:** All screenshots are exactly 2000×1250 pixels (as required by Raycast)
- **File Format:** PNG with optimization
- **Naming Convention:** Lowercase kebab-case (e.g., `advanced-glb-1.png`)

## Notes

- All screenshots are actual captures of the extension running in Raycast
- Images have been resized to meet Raycast's exact dimension requirements (2000×1250 pixels)
- Screenshots demonstrate the extension's key features: compression, optimization, and inspection
- The screenshots follow Raycast's extension guidelines for metadata presentation
